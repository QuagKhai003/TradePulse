# web — TradePulse frontend (Layer 1)

Next.js (App Router, JS) + d3-geo choropleth. Reads the ETL snapshot — no DB binding.

## Run (localhost)
```bash
# 1) build the data snapshot first (from repo root)
cd etl && python -m tradepulse_etl        # writes web/public/data/snapshot.json

# 2) run the app
cd ../web && npm install                  # first time only
npm run dev                               # -> http://localhost:3200
```
Port is **3200** (3000 is often taken). `?lang=en` toggles English (Vietnamese is default).

## What renders (plan §7.1)
- SSR world choropleth colored by demand signal band (covered markets only).
- Market tiles: value + YoY band + Vietnam import share + honest period/published labels.
- Signal feed: moderate+ signals, ranked by severity then value.
- SAMPLE-DATA banner while the source is the offline fixture.

## Layout
```
app/
  page.js            # Layer-1 landing (server component)
  layout.js          # root, VN default
  globals.css
  components/        # WorldMap, MarketTile, SignalFeed
  lib/               # snapshot loader, format/band colours, i18n strings
public/data/         # snapshot.json (generated by the ETL, gitignored)
```

## Notes
- **Seam:** the app reads `public/data/snapshot.json`; production can swap to a live API with no
  component change (contract in `docs/DATA_MODEL.md`).
- **Golden Rule:** inform-never-match — the UI shows data only; no parties, no contacts.
- Rebuild data anytime by re-running the ETL; refresh the page (read at request time).
